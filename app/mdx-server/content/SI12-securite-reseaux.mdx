# Sécurité des Réseaux Informatiques

<ExerciseCard title="Découvrir la sécurité des réseaux">

Dans cette leçon, tu vas découvrir comment protéger les réseaux informatiques contre les menaces et les attaques, et comprendre les principes fondamentaux de la cybersécurité.

Voici ce que tu vas apprendre :
- Les principales menaces qui pèsent sur les réseaux informatiques
- Les différents mécanismes de protection
- Les bonnes pratiques pour sécuriser un réseau
- Comment réagir face à un incident de sécurité

        <p>Ces programmes sont conçus pour s'infiltrer dans les systèmes et causer des dommages :</p>
        <ul>
          <li><strong>Virus</strong> : Se propagent en s'attachant à d'autres programmes</li>
          <li><strong>Vers</strong> : Se propagent automatiquement sans intervention humaine</li>
          <li><strong>Chevaux de Troie</strong> : Se présentent comme des logiciels légitimes</li>

        <p>Ces attaques visent à rendre un service indisponible :</p>
        <ul>
          <li>En surchargeant le réseau de requêtes</li>
          <li>En épuisant les ressources des serveurs</li>
          <li>Souvent menées depuis plusieurs sources simultanément (DDoS)</li>
        </ul>
        
        <h4>4. Hameçonnage (phishing)</h4>
        <p>Technique qui consiste à tromper l'utilisateur pour obtenir des informations sensibles :</p>
        <ul>

          <li>Sites web contrefaits</li>
          <li>Messages incitant à une action urgente</li>

        <p>Capture des données qui transitent sur le réseau :</p>
        <ul>
          <li>Particulièrement risqué sur les réseaux Wi-Fi non sécurisés</li>
          <li>Peut révéler des mots de passe et informations confidentielles</li>
        </ul>

        <p>L'attaquant s'intercale entre deux parties qui communiquent :</p>
        <ul>
          <li>Il peut lire, modifier ou injecter des messages</li>
          <li>Les parties légitimes pensent communiquer directement entre elles</li>

    content: (
      <div>
        <p>Pour contrer ces menaces, plusieurs mécanismes de protection peuvent être mis en place :</p>
        
        <h4>1. Pare-feu (Firewall)</h4>
        <p>Le <strong>pare-feu</strong> est la première ligne de défense d'un réseau :</p>
        <ul>
          <li>Il filtre le trafic entrant et sortant selon des règles définies</li>
          <li>Il bloque les connexions non autorisées</li>
          <li>Il peut être matériel (boîtier dédié) ou logiciel (installé sur un ordinateur)</li>
        </ul>
        
        <div style={{ backgroundColor: '#f8f9fa', padding: '15px', borderRadius: '5px', marginBottom: '20px' }}>
          <p style={{ fontWeight: 'bold' }}>Exemple de règle de pare-feu :</p>
          <p>Autoriser le trafic HTTP (port 80) et HTTPS (port 443) entrant vers le serveur web, mais bloquer tout autre trafic entrant vers ce serveur.</p>
        </div>
        
        <h4>2. Systèmes de détection et de prévention d'intrusion (IDS/IPS)</h4>
        <p>Ces systèmes surveillent le réseau pour détecter les activités suspectes :</p>
        <ul>
          <li><strong>IDS</strong> (Intrusion Detection System) : Détecte et alerte</li>
          <li><strong>IPS</strong> (Intrusion Prevention System) : Détecte et bloque automatiquement</li>
        </ul>
        
        <h4>3. Réseaux privés virtuels (VPN)</h4>
        <p>Les <strong>VPN</strong> créent un tunnel sécurisé pour les communications :</p>
        <ul>
          <li>Ils chiffrent les données qui transitent sur le réseau</li>
          <li>Ils masquent l'adresse IP réelle</li>
          <li>Particulièrement utiles pour se connecter à distance de manière sécurisée</li>
        </ul>
        
        <h4>4. Chiffrement des données</h4>
        <p>Le <strong>chiffrement</strong> rend les données illisibles sans la clé appropriée :</p>
        <ul>
          <li><strong>HTTPS</strong> : Sécurise les connexions web</li>
          <li><strong>TLS/SSL</strong> : Protocoles de sécurisation des communications</li>
          <li><strong>Chiffrement de bout en bout</strong> : Seuls l'expéditeur et le destinataire peuvent lire les messages</li>
        </ul>
        
        <h4>5. Authentification et contrôle d'accès</h4>
        <p>Ces mécanismes vérifient l'identité des utilisateurs et limitent leurs droits :</p>
        <ul>
          <li><strong>Mots de passe</strong> : Première ligne de défense</li>
          <li><strong>Authentification à deux facteurs (2FA)</strong> : Combine deux éléments (mot de passe + code temporaire)</li>
          <li><strong>Biométrie</strong> : Utilise des caractéristiques physiques (empreinte digitale, reconnaissance faciale)</li>
          <li><strong>Contrôle d'accès basé sur les rôles</strong> : Attribue des droits selon la fonction de l'utilisateur</li>
        </ul>
        
        <h4>6. Segmentation du réseau</h4>
        <p>Diviser le réseau en zones distinctes pour limiter la propagation des menaces :</p>
        <ul>
          <li><strong>VLAN</strong> (Virtual Local Area Network) : Sépare logiquement le réseau</li>
          <li><strong>DMZ</strong> (Zone démilitarisée) : Zone tampon pour les serveurs accessibles depuis l'extérieur</li>
        </ul>

<ExerciseCard title="Exercice pratique : Évaluer et renforcer la sécurité d'un réseau">

Maintenant que tu comprends mieux la sécurité des réseaux, il est temps de mettre en pratique tes connaissances !

### Exercice 1 : Audit de sécurité basique

Réalise un mini-audit de sécurité sur ton réseau domestique ou celui de ton école (avec autorisation) :

1. Identifie les équipements connectés au réseau.
2. Vérifie les paramètres de sécurité du routeur/point d'accès Wi-Fi.
3. Note les vulnérabilités potentielles que tu observes.
4. Propose des améliorations pour renforcer la sécurité.

### Exercice 2 : Création d'une politique de mots de passe

Élabore une politique de mots de passe pour un petit réseau (famille ou petite entreprise) :

1. Définis les critères de complexité des mots de passe.
2. Établis des règles pour le renouvellement des mots de passe.
3. Propose une stratégie pour la gestion des mots de passe (gestionnaire de mots de passe, etc.).
4. Explique comment mettre en place l'authentification à deux facteurs.

### Exercice 3 : Plan de réponse aux incidents

Crée un plan de réponse simple pour les incidents de sécurité suivants :

1. Infection par un malware
2. Perte ou vol d'un appareil contenant des données sensibles
3. Suspicion d'accès non autorisé au réseau
4. Attaque par hameçonnage (phishing)

Pour chaque scénario, indique :
- Les actions immédiates à entreprendre
- Les personnes à contacter
- Les étapes pour revenir à la normale
- Les mesures pour éviter que cela ne se reproduise

### Exercice 4 : Segmentation de réseau

Imagine que tu dois concevoir un réseau sécurisé pour une petite entreprise avec :
- Des postes de travail pour les employés
- Des serveurs internes contenant des données sensibles
- Des appareils IoT (caméras, thermostats intelligents)
- Un réseau Wi-Fi pour les visiteurs

Propose un schéma de segmentation du réseau en expliquant :
1. Comment tu séparerais les différents types d'appareils
2. Quelles règles de pare-feu tu mettrais en place
3. Comment tu gérerais les accès entre les segments
