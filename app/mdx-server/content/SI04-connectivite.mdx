# La Connectivité

<ExerciseCard title="Découvrir les technologies de connexion">

Dans cette leçon, tu vas découvrir comment les systèmes informatiques communiquent entre eux grâce à différentes technologies de connectivité.

Voici ce que tu vas apprendre :
- Les différents types de connexions (filaires et sans fil)
- Les réseaux informatiques et leur fonctionnement
- Les protocoles qui permettent la communication
- Les enjeux de sécurité liés à la connectivité

</ExerciseCard>

## Exercice 1: Identification des technologies

Pour chaque situation ci-dessous, indique quelle technologie de connectivité serait la plus appropriée et pourquoi :
1. Connecter un ordinateur fixe à Internet avec une connexion stable et rapide
2. Relier un smartphone à des écouteurs sans fil
3. Connecter un ordinateur portable à Internet dans un café
4. Transférer de grandes quantités de données entre deux serveurs dans un centre de données

<details>
  <summary>Solution</summary>
  <p>1. Ethernet : offre une connexion stable, rapide et fiable pour un équipement fixe</p>
  <p>2. Bluetooth : idéal pour les périphériques audio, faible consommation d'énergie, portée suffisante</p>
  <p>3. Wi-Fi : disponible dans les lieux publics, permet la mobilité, débit suffisant pour la navigation</p>
  <p>4. Fibre optique : permet des débits très élevés nécessaires pour transférer de grandes quantités de données</p>
</details>

## Exercice 2: Adressage IP

1. Identifie si les adresses IP suivantes sont valides en IPv4 :
   - 192.168.1.256
   - 10.0.0.1
   - 172.16.254.1
   - 256.0.0.1

2. Détermine si les adresses suivantes sont des adresses IP privées ou publiques :
   - 192.168.0.1
   - 8.8.8.8
   - 10.0.0.1
   - 172.16.0.1

<details>
  <summary>Solution</summary>
  <p>1. Validité des adresses IPv4 :</p>
  <ul>
    <li>192.168.1.256 : Non valide (les nombres doivent être entre 0 et 255)</li>
    <li>10.0.0.1 : Valide</li>
    <li>172.16.254.1 : Valide</li>
    <li>256.0.0.1 : Non valide (les nombres doivent être entre 0 et 255)</li>
  </ul>
  <p>2. Adresses privées vs publiques :</p>
  <ul>
    <li>192.168.0.1 : Privée (plage 192.168.0.0 - 192.168.255.255)</li>
    <li>8.8.8.8 : Publique (serveur DNS de Google)</li>
    <li>10.0.0.1 : Privée (plage 10.0.0.0 - 10.255.255.255)</li>
    <li>172.16.0.1 : Privée (plage 172.16.0.0 - 172.31.255.255)</li>
  </ul>
</details>

## Exercice 3: Diagnostic réseau

Pour chacun des problèmes suivants, propose une solution :
1. Tu ne peux pas accéder à Internet, mais ton ordinateur est bien connecté au réseau Wi-Fi
2. La connexion Wi-Fi est très lente dans certaines pièces de la maison
3. Tu ne peux pas te connecter à un site web spécifique, mais les autres sites fonctionnent

<details>
  <summary>Solution</summary>
  <p>1. Problème d'accès à Internet malgré connexion Wi-Fi :</p>
  <ul>
    <li>Vérifier si le routeur est connecté à Internet (voyants)</li>
    <li>Redémarrer le modem et le routeur</li>
    <li>Vérifier si l'ordinateur a bien obtenu une adresse IP (ipconfig/ifconfig)</li>
    <li>Tester la connexion avec la commande ping (ping 8.8.8.8)</li>
    <li>Vérifier si le DNS fonctionne (ping www.google.com)</li>
  </ul>
  <p>2. Connexion Wi-Fi lente dans certaines pièces :</p>
  <ul>
    <li>Repositionner le routeur dans un emplacement plus central</li>
    <li>Installer un répéteur Wi-Fi ou un système mesh</li>
    <li>Vérifier les interférences potentielles (autres réseaux, appareils électroniques)</li>
    <li>Changer de canal Wi-Fi ou passer à la bande 5 GHz si disponible</li>
  </ul>
  <p>3. Accès impossible à un site spécifique :</p>
  <ul>
    <li>Vider le cache du navigateur</li>
    <li>Vérifier si le site est accessible depuis un autre appareil ou réseau</li>
    <li>Essayer un autre navigateur</li>
    <li>Vérifier si le site n'est pas bloqué par un pare-feu ou un contrôle parental</li>
    <li>Utiliser un service comme Down Detector pour vérifier si le site est en panne</li>
  </ul>
</details>

## Défis créatifs

1. **Cartographie réseau** : Dessine un schéma du réseau de ton domicile ou de ton école, en identifiant tous les appareils connectés et les technologies de connexion utilisées.

2. **Analyse de performances** : Mesure et compare les performances (débit, latence) de différentes connexions auxquelles tu as accès (Wi-Fi, 4G, Ethernet, etc.) à l'aide d'outils comme Speedtest.net.

3. **Configuration réseau** : Si tu as accès à un routeur, explore son interface d'administration et identifie les différents paramètres de configuration (SSID, canal Wi-Fi, contrôle parental, redirection de ports, etc.).
