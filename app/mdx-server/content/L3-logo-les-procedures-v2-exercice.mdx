**Instructions (La manière "longue") :**

Essayez de réaliser ce dessin en tapant les commandes suivantes directement dans la ligne de commande de Logo. Appuyez sur `Entrée` après chaque ligne.

```logo
VE  ; Efface l'écran et remet la tortue au centre (juste au cas où)

; Dessiner le premier carré
REPETE 4 [ AV 50 TD 90 ]

; Se déplacer pour le deuxième carré (sans dessiner)
PU       ; Lève le crayon
AV 70    ; Avance (50 pour le côté + 20 pour l'espace)
PD       ; Pose le crayon

; Dessiner le deuxième carré
REPETE 4 [ AV 50 TD 90 ]

; Se déplacer pour le troisième carré (sans dessiner)
PU       ; Lève le crayon
AV 70    ; Avance (50 pour le côté + 20 pour l'espace)
PD       ; Pose le crayon

; Dessiner le troisième carré
REPETE 4 [ AV 50 TD 90 ]
```

**Réfléchissons un instant :**

1.  Avez-vous trouvé cela un peu **long et répétitif** à taper ?
2.  Avez-vous remarqué que la commande `REPETE 4 [ AV 50 TD 90 ]` est utilisée **exactement de la même manière** trois fois ?
3.  Imaginez que vous deviez dessiner **10 carrés** au lieu de 3. Serait-ce pratique de tout retaper ?
4.  Et si, après avoir tout tapé, vous décidiez que les carrés devraient avoir une taille de **80 pas** au lieu de 50 ? Combien de lignes de code devriez-vous modifier ? (Réponse : 3 lignes !)

Ce genre de répétition est exactement le problème que les **procédures** viennent résoudre ! Elles nous permettent de donner un nom simple (comme `CARRE50`) à une série d'instructions et de réutiliser ce nom autant de fois que nécessaire.

Maintenant que vous avez ressenti le besoin, découvrons comment créer et utiliser ces fameuses procédures.
